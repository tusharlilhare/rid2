How the Web Works


1. User


The user initiates a request by entering a URL in the browser.

2. Browser


The browser processes the request and communicates with the search engine or DNS to locate the server.

Search Engine


Helps locate the domain name and sends a request to the DNS.

→

DNS


Translates the domain name into an IP address (e.g., 192.333.139.003).

→

Server

Hosts the website files and sends the response back to the browser.


3. Website


The browser displays the website using:

  • HTML (structure)
  • CSS (style/look)
  • JavaScript (logic)

Frontend frameworks (e.g., Angular, React, Vue.js) and backend frameworks (e.g., Django, Express.js) enhance functionality.


Web


  • The Web (short for World Wide Web) is a global system of interconnected computer networks that use the Internet Protocol Suite to share and access information
  • It allows users to interact and retrieve information using web browsers.
  • It can also be seen as a virtual directory on web servers or a portion of the broader Internet that primarily focuses on hypertext and multimedia resources.

Site


  • A site (short for website) is a collection of web pages hosted on a web server and accessed through a unique domain or URL.
  • It represents a specific location on the internet, often dedicated to a particular purpose or theme (e.g., news, commerce, or education).

Page


  • A page refers to an individual document or resource on the web, accessible via a specific URL.
  • It is part of a larger website and contains structured content such as text, images, videos, or interactive elements.

Web Page


  • A web page is a single hypertext document that is part of the World Wide Web (WWW).
  • It contains multimedia content and links, allowing navigation beyond the displayed content.
  • The term "hyper" (from the Greek root meaning "beyond") reflects the page's capability to connect to other documents or media through hyperlinks.
